# Wane — лимиты Claude в меню-баре macOS

Wane показывает в строке меню-бара macOS:
- остаток лимита текущей сессии (окно 5 часов);
- остаток недельного лимита;
- последнюю использованную модель Claude Code;
- при приближении к лимиту (≤30%) — оценку «хватит ещё ~N минут при этом темпе».

Публичная бета. Бесплатно на этом этапе.

## Установка

1. Скачайте `Wane-1.0.0.zip` из [Releases](../../releases), распакуйте.
2. Перетащите `Wane.app` в `/Applications`.
3. Запустите. macOS покажет предупреждение «нельзя проверить разработчика» —
   это ожидаемо: приложение пока распространяется без платной Apple
   Developer подписи ($99/год), которую есть смысл оформлять только после
   подтверждения спроса на бету.

   **Как открыть:** System Settings → Privacy & Security → в самом низу
   найдите строку про Wane.app → **Open Anyway**. Делается один раз.

4. При первом запуске приложение попросит доступ к Keychain — см. ниже,
   зачем это нужно.

## Зачем доступ к Keychain

Wane читает OAuth-токен, который уже сохраняет сам Claude Code в Keychain
(сервис `Claude Code-credentials`), — тем же токеном, что использует
команда `/usage`. Токеном Wane обращается к
`api.anthropic.com/api/oauth/usage`, чтобы получить процент остатка лимита.

- Токен никуда, кроме `api.anthropic.com`, не отправляется.
- Токен нигде не логируется и не сохраняется в открытом виде.
- Wane не собирает телеметрию и не отправляет данные на сторонние серверы.

Диалог Keychain появится один раз — нажмите **Always Allow**.

## Автообновления

Wane проверяет обновления через Sparkle (не чаще раза в сутки) и предложит
установить новую версию, когда она выйдет. Вручную — пункт меню
«Проверить обновления…».

## Как это устроено (коротко)

Официального публичного API для лимитов подписки нет. Wane использует тот
же недокументированный endpoint, что и сама команда `/usage` в Claude Code.
Это осознанный риск: endpoint может измениться без предупреждения.

## Требования

macOS 13+, Apple Silicon (arm64).
